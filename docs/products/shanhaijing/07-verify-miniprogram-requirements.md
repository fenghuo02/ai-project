# 「山海有灵 · 验真」小程序 · 功能与合规需求文档

> 配套 `06-anti-counterfeit.md` 三级防伪体系，提供消费者验真与运营溯源。
> 定位：**轻量验真工具**，不是商城、不是社区。一切以「查真伪 + 看收藏信息」为边界。
>
> ⚠️ 合规优先级最高：**免登录即可验真**、**无功效宣称**、**个人信息最小必要**。本文为需求文档，开发与上线前须经法务 + 平台审核。

---

## 一、背景与目标

1. 解决仿冒、保护限量收藏价值（接 06 · L1/L2/L3）。
2. 消费者扫码 / NFC / 输入验证码 → 秒级返回真伪 + 编号 + 批次。
3. 运营端可追溯生产批次、查验记录、异常预警。

## 二、用户角色

| 角色 | 权限 | 说明 |
|---|---|---|
| 消费者（游客） | **免登录**验真查询 | 默认无需授权任何个人信息 |
| 收藏者（可选注册） | 绑定数字证书 / 转移登记 | 仅 GK 收藏款；自愿、可注销 |
| 运营后台 | 防伪码生成、批次管理、查验统计、预警 | 内部 RBAC 权限分级 |
| 厂端录入（受控） | 批次与码段写入 | 仅限授权账号、操作留痕 |

## 三、功能需求

### 3.1 消费者端（核心）
- F1 **三种验真入口**：扫码（二维码）/ NFC 轻触 / 手动输入 16 位验证码。
- F2 **验真结果页**：真伪结论 + SKU 名称 + 限量编号 + 批次 + 「本码已被查询 N 次 / 首次查询时间」。
- F3 **异常提示**：二次/高频查询、异地集中查询 → 显示「疑似翻包/仿冒，请通过官方渠道核实」。
- F4 **收藏信息展示**（仅 GK）：限量号、发行量、发行日、数字证书编号（只读）。
- F5 **官方信息位**：真品购买渠道、客服入口、防伪说明（**不放功效宣传**）。

### 3.2 收藏者端（可选，仅 GK）
- F6 数字证书绑定（NFC UID ↔ 证书号），自愿提供手机号 + 单独同意。
- F7 收藏转移登记（二手流转），双方确认。
- F8 账号注销 + 数据删除入口。

### 3.3 运营后台
- F9 防伪码批量生成（唯一、不可预测、防枚举）与码段管理。
- F10 批次 / SKU / 限量编号录入与绑定。
- F11 查验数据看板：查询量、重复查询 TOP、地域分布、异常告警。
- F12 NFC UID 写入与状态管理（启用/作废）。
- F13 操作审计日志（谁、何时、改了什么）。

## 四、非功能需求

- 性能：验真查询 P95 < 1s；支持大促峰值并发。
- 可用性：验真接口 SLA ≥ 99.9%；离线降级提示。
- 安全：验证码加盐哈希存储；防枚举（限频 + 验证码段不连续）；接口防刷（限流 + 风控）；HTTPS 全链路。
- 兼容：微信小程序为主（可扩展支付宝小程序 / H5）。

## 五、合规需求（重点）⭐

### 5.1 个人信息保护（PIPL）
- **默认免登录验真**：游客查询**不收集**任何个人信息（不强制授权手机号/位置/微信资料）。
- **最小必要**：仅 GK 绑定证书等确有必要时，才收集手机号，且**单独勾选同意**、说明用途与期限、**可撤回 / 可删除 / 可注销**。
- **隐私政策 + 个人信息收集清单**：首次使用弹出，明确收集项、目的、存储期限、第三方共享情况。
- **儿童信息**：玩具用户可能为未成年人，不主动收集儿童信息；如涉及，需监护人同意机制。

### 5.2 平台与资质合规
- 微信小程序：按经营内容选**正确服务类目**；完成小程序备案 / ICP（按最新要求）。
- 遵守《微信小程序运营规范》《平台运营规则》；不得诱导分享、不得违规收集。
- 主体：以 **B 文创公司** 作为小程序注册与备案主体（见 §十八）。

### 5.3 内容与广告合规
- 验真页**禁止**任何功效宣称（招财/辟邪/改运/护身/保平安）与升值/投资暗示（§11.4 广告法）。
- 文案仅限「真伪、编号、批次、收藏信息、购买与客服」。
- 涉及 AI 生成图样的展示，保留「含 AI 生成」标识（§11.2）。

### 5.4 数据安全（数据安全法 / 网络安全法）
- 数据分级：验证码、UID、查验日志按内部密级管理；敏感字段加密存储。
- 最小权限 + 审计；定期安全评估与漏洞修复。
- 第三方 SDK 清单与告知（扫码、NFC、统计、风控等），逐一在隐私政策列明并取得同意。

### 5.5 数字证书 / 存证（如启用）
- 若用区块链/数据库存证：仅作**真伪与收藏信息**记录，**不得**宣传金融、投资、升值、可交易增值属性。
- 存证服务商须合规、数据可审计。

### 5.6 跨境（如涉及海外）
- 海外账号引流来的境外用户验真，若涉及个人信息出境，按 PIPL 出境规则处理（默认验真免登录可规避）。

## 六、数据字典（核心实体）

| 实体 | 关键字段 |
|---|---|
| 防伪码 Code | code_id, 哈希值, 关联SKU, 批次, 状态, 首查时间, 查询次数 |
| NFC 标签 | uid, 关联证书号, 状态(启用/作废) |
| SKU | sku_id, 名称, 谱系(神兽/凶兽), 是否限量 |
| 批次 Batch | batch_id, SKU, 数量, 生产日期, 工厂 |
| 限量证书 | cert_no, 编号x/总量, 发行日, 持有登记(可选) |
| 查验记录 | code_id, 时间, 地域(粗粒度), 结果 |

## 七、验真流程（文字版）

1. 用户扫码/NFC/输码 → 2. 校验码合法性与状态 → 3. 命中：返回真伪+编号+批次+查询次数；未命中/异常：返回风险提示 → 4. 记录一条查验日志（不含个人身份信息） → 5.（可选）GK 用户自愿绑定证书（单独同意）。
