# Second Brain System · 知识捕获 → 整理 → 提炼 → 输出

> 状态：v1.0 · 2026-05-30
> 触发：v3+v4+v5 系统要求 7 推/日 + 1 article/周 + 1 video/周 + Spaces + cross-platform。**没有 idea capture 系统 = 灵感日 vs 干涸日震荡 = 长期不可持续**。Karpathy / 大量大 V 实践已证明 second brain 是创作者必备 infrastructure。

## 一、为什么必须建 second brain

### 1.1 Creator 是 knowledge worker（输入 → 输出）

```
你的工作 = 把"读到的 + 观察到的 + 想到的" 转成 "受众消费"
  · 读 filings / earnings calls → 1 个 take
  · 看 twitter timeline → 1 个 reply
  · 想到对比 → 1 张 comparison table

如果输入 → 输出 没系统:
  · 灵感日: 7 推都灵感来 = 输出涌泉
  · 干涸日: 0 灵感 = 干瞪眼
  · 平均 = 焦虑 + burnout

如果有 second brain:
  · 灵感日: 写 idea → idea pool
  · 干涸日: 从 pool 取 → 仍能输出
  · 平均 = 稳定 + 长期复利
```

### 1.2 行业证据（2026-05 联网验证）

- [Karpathy Second Brain in Obsidian](https://medium.com/@theo-james/karpathy-second-brain-in-obsidian-exact-structure-e9069a62f025)：rapidly capture ideas → distill into reusable atoms → connect into network
- [Obsidian 2026 增长 22% YoY 突破 1.5M users](http://www.nxcode.io/resources/news/obsidian-ai-second-brain-complete-guide-2026)：local-first + Markdown 是 AI-powered 第二大脑理想基础
- [Tiago Forte CODE 框架](https://www.automateed.com/second-brain-systems-for-content-creators)：Capture → Organize → Distill → Express
- [Saner.ai 2026 best app review](https://blog.saner.ai/second-brain-app/)：Notion / Obsidian / Capacities / Craft 是 2026 top picks
- [aimaker AI Second Brain](https://aimaker.substack.com/p/llm-wiki-obsidian-knowledge-base-andrej-karphaty)：Karpathy LLM Wiki pattern 已成 creator 标准

> 内容已合规化转述。

### 1.3 你目前的"散乱信息"

```
当前情况:
  · X 看到 100 推 → 30 个有用 → 1 个被 reply → 99 忘了
  · 读 filings 30 分钟 → 5 个 insight → 输出 1 推 → 4 个忘了
  · 月底总觉得"上月看了好多但都没用上"
  · Sunday Long Read 写到一半发现"我之前想过这个但忘了"

→ 浪费的 80% 输入是 second brain 应该 capture 的
```

## 二、CODE 框架（Capture → Organize → Distill → Express）

### 2.1 整体流程

```
INPUT (输入)
  ↓
CAPTURE (5 min/天)
  · 想法 / quote / 数字 / 链接 都进 inbox
  · 不分类 / 不评判 / 只 capture
  ↓
ORGANIZE (20 min/周日)
  · Inbox → topic folders
  · 标签 + 时间锚
  ↓
DISTILL (10 min/周日)
  · 同主题 ideas 合并成 "atom"
  · 1 atom = 1 reusable take
  ↓
EXPRESS (持续)
  · Daily prompt 跑时 atom pool 是输入
  · Sunday Long Read 主题 distill 自 atoms
  · Newsletter 反映 weekly patterns
  ↓
OUTPUT (输出)
```

### 2.2 4 个阶段时间分配

| 阶段 | 频率 | 时间 | 工具 |
|---|---|---|---|
| Capture | 每天 5 min（早晚 each 2.5 min）| 5 min/天 = 35 min/周 | 手机 Notion / Obsidian quick capture |
| Organize | 每周日 | 20 min | Notion / Obsidian web |
| Distill | 每周日 | 10 min | 同上 |
| Express | 持续（融进现有 daily SOP）| 0 额外（已 covered）| Grok / Kimi prompts |

**总额外时间**: ~65 min/周。**节省**: 每周 1-2 hr"我应该写啥" 焦虑时间 + 长期复利。

## 三、CAPTURE · 5 类该 capture 的

### 3.1 必 capture 的 5 类

```
1. 数字 / 数据点
   · 任何看到的具体数字 + 来源 + 日期
   · 例: "anthropic $30B annualized as of 5/29/2026"

2. Quote (别人 say 的)
   · 大 V / CEO 说的"奇怪 / 反常识 / 重要"句
   · 例: "@elonmusk 5/22: 'robotaxi by year end. !!'"
   · → 加进 Musk Tweet Decoder Dictionary 

3. Connection (我想到的关联)
   · 两个看似无关的事如何相关
   · 例: "anthropic $30B 6.7× yoy = nvidia 2023 trajectory"

4. Take (我的观察 / 立场)
   · 一句话观点
   · 例: "fsd subs growing 6× faster than auto = software co transition"

5. Question (我不知道的)
   · 触发未来研究
   · 例: "why does anthropic keep beating openai on coding benchmarks?"
```

### 3.2 不 capture 的

```
❌ 别人完整文章（链接进 raindrop / pocket，不进 second brain）
❌ Newsletter 完整 issue（链接 only）
❌ 长视频 transcript（不必要）
❌ 私事 / 情绪（不在工作 system 里）
❌ Politics / 宗教 / 个人喜好（违反账号 brand）
```

### 3.3 Capture 时机

```
推荐 4 个 capture moments:

1. 早 06:30-06:35 起床后 5 min
   · 看 X feed 任何"哦" 的 → capture
   · 看 push notification 任何 important → capture

2. 工作日中午午餐前 5 min
   · 早上看到 / 想到的 → capture

3. 晚 22:00-22:05 睡前
   · 当天最后 capture
   · 标"tomorrow's prompt 用得到" if any

4. Ad-hoc: 任何"我想用这个"瞬间
   · 手机 quick capture (Notion app / Obsidian Mobile)
   · 30 秒 input - 不超过
```

### 3.4 Capture 模板（每条 inbox item）

```
[Notion / Obsidian database: Inbox]

| Field | Value |
|---|---|
| Title | 一句话总结（你 read it 5 月后能识别）|
| Content | 1-3 句完整 capture |
| Source | URL / 推 ID / "thinking" |
| Date | YYYY-MM-DD |
| Topic tags | tesla, anthropic, AI race, markets, musk timeline, etc |
| Type | data / quote / connection / take / question |
| Status | inbox (default) |
| Notes | 可选 |
```

## 四、ORGANIZE · 周日 20 min

### 4.1 流程

```
Step 1 (5 min) · Inbox audit
  · 看上周 captures (~30-50 items)
  · 删 duplicates
  · 删明显无用（事后看不重要）

Step 2 (10 min) · Topic 分类
  · 每个 capture → 1-3 个 topic folders:
    · tesla
    · anthropic
    · openai
    · xai
    · nvidia
    · spacexai
    · neuralink
    · boring
    · ai race
    · markets / ipo
    · musk timeline
    · cultural observation
    · personal CRM
    · build in public

Step 3 (5 min) · Tag 时间锚
  · 时效性 tag:
    · "fresh" (this week)
    · "monthly" (this month)
    · "evergreen" (timeless)
  · 优先级 tag:
    · "use next week"
    · "use this month"
    · "long-term reference"
    · "discard candidate"
```

### 4.2 Topic Folder 模板（Notion）

```
[Database: tesla]
- 5/14 · Q1 deliveries 358k missed (data) [fresh]
- 5/22 · Optimus 27 DOF hand demo (data) [monthly]
- 5/27 · FSD subs trajectory note (take) [evergreen]
- 5/30 · Musk q "robotaxi !!" decoder (quote) [fresh]
- ...

链接关系:
  · 5/22 Optimus + 5/27 FSD note = "tesla 2026 = robotics + software"
  · 5/14 q1 miss + 5/27 FSD = "auto miss but software win"
```

## 五、DISTILL · 10 min/周日

### 5.1 流程

```
看每个 topic folder (1 min/folder × 10 folders):

Step 1: 找 connections
  · 哪 2-3 个 captures 关联?
  · 写 1 个 connection note (atom)

Step 2: 提炼 reusable take
  · 把 connection 写成 1 句话
  · 例:
    "tesla 2026 transition: auto deliveries miss + fsd subs +51% yoy
     = 6× faster software growth = software co transition by Q4"

Step 3: 标注用途
  · "next monday empire map" / "next sunday long read" / "next reading list" / "evergreen"
```

### 5.2 Atom 模板

```
[Database: Atoms]

| Field | Value |
|---|---|
| Title | 一句话 take |
| Connection | 引用 2-3 个 capture IDs |
| Topic | tesla / anthropic / etc |
| Type | take / framework / data point |
| Use for | next monday / next sunday / evergreen |
| Voice tag | 哪个招牌句式适合 ([signature-voice.md §三](./signature-voice.md)) |
| Status | unused / used (date) |
```

### 5.3 Atom 示例

```
TITLE: "tesla 2026 = software co transition by q4"

CONNECTION:
- inbox_5/14 · q1 deliveries 358k missed
- inbox_5/27 · fsd subs 1.28M +51% yoy
- inbox_5/22 · optimus 27 DOF demo

TOPIC: tesla

TYPE: framework

USE FOR: next monday empire map (week 23)

VOICE TAG: "the part i keep thinking about: ..."

STATUS: unused

---

EXPRESSION DRAFT:
"the part i keep thinking about for tesla:
auto deliveries missed by 7k. fsd subs +51% yoy at 1.28M.
auto growth ~5%/yr. software growth 51%/yr.
that's not a 'mixed quarter'. that's a phase transition.
software co by q4."
```

## 六、EXPRESS · 把 atoms 转成内容

### 6.1 Daily SOP 整合

```
每天早上 06:30 跑 Grok §14 prompt:
  · 旧（v3-v5）: Grok 联网搜过去 24h 新闻 + 输出 7 推
  · 新（v6）: 加一步: 你 paste 1-3 个最适合今天的 atoms
            Grok 把 atom 编织进 7 推

例:
  Daily prompt input:
  "Today is week 23 monday. 
   Atoms to use:
   - 'tesla 2026 = software co transition by q4'
   - 'anthropic 32× revenue is conservative'
   Output 7 推 with these atoms incorporated."

  Grok 输出: 1 main (用 tesla atom) + 5 humor + 1 expansion (anthropic atom)
```

### 6.2 Sunday Long Read 主题 distill

```
周日 19:00 写 Long Read 时:
  · 看本周 distilled atoms (~5-10)
  · 选 1 个最 dense / 最有数据的
  · 围绕这个 atom 写 1500-3000 字 deep dive

→ Long Read 不是临场憋出来，是 1 周累积 atoms 的成果
→ 质量稳定，不靠灵感
```

### 6.3 Reading List atoms

```
Friday Reading List:
  · 来源: 本周读到的"connections" capture
  · 5 篇 = 5 个本周最有 spark 的 connection notes
  · 每个 connection 都自然包含 1-2 个其他人的引用
```

### 6.4 Newsletter atoms

```
Sunday Newsletter:
  · 5-bullet TL;DR = 本周 5 个最强 atoms
  · "Number of the week" = 本周最 actionable data point
  · Each section pulls from atom pool
```

## 七、Tool Stack · Notion vs Obsidian vs 手机记事本

### 7.1 决策表

| 维度 | Notion | Obsidian | 手机记事本 |
|---|---|---|---|
| 上手 | 简单 | 中等 | 极简 |
| 跨设备 sync | 自带 | iCloud / Sync 付费 | 自带 |
| 链接 / backlinks | 弱 | 强 | 无 |
| AI 集成 | Notion AI | Smart Connections plugin | 无 |
| 价格 | $0-10/月 | $0 (Sync $4-8/月) | $0 |
| 推荐 | M1-M2 起步 | M3+ 升级（如果觉得 Notion 不够）| 临时备份 |

### 7.2 推荐 stack（按阶段）

```
M1 (now):
  Notion (free plan)
    · Inbox database
    · Topic databases (10-15 tags)
    · Atoms database
  手机 Notion app (quick capture)
  
  Total cost: $0
  Total time: 5 min/天 + 30 min/周日 = ~65 min/周

M3+ (M3 起):
  Notion (Plus $10/月) + Obsidian (free)
    · Notion: Personal CRM + bookkeeping (visual)
    · Obsidian: Inbox + Atoms (text-heavy + AI plugin)
  
  Total cost: $10/月
  Total time: 同上 65 min/周

M5+ (M5 起):
  Notion + Obsidian + Otter.ai (audio capture)
    · Otter: Spaces 录音 → text → atoms
    · Obsidian: AI plugin self-query
  
  Total cost: $30-40/月
```

### 7.3 不需要的工具

```
❌ Roam Research (太贵 + 学曲线高)
❌ RemNote (spaced repetition 不适合 creator)
❌ Logseq (类似 Obsidian 但 ecosystem 弱)
❌ Capacities (新 + 数据迁移风险)
❌ Saner.ai (新 + 不必要)
```

## 八、Atoms 数量目标

### 8.1 阶段目标

| 阶段 | Atom 库存 | 月度新增 | 用过 % |
|---|---|---|---|
| M1 (now) | 0 → 30 | 30 | 50% |
| M2 | 30 → 80 | 50 | 60% |
| M3 | 80 → 200 | 60 | 70% |
| M5+ | 500-1000 | 80 | 70%（持续）|

```
关键: 不是积越多越好。
  · 70% 用过率 = 健康
  · < 50% 用过率 = inbox-organize-distill 流程有问题
  · > 90% 用过率 = capture 不够（错过 input）
```

### 8.2 Atom 退场

```
每月最后一天 retro 清理:
  □ 6 个月没 use 的 → "evergreen" 或 "discard"
  □ 错的 / 过时的 → discard
  □ Use 5+ 次的 → "core atom" (永远保留)
  
保持 atom pool 干净 + actionable
```

## 九、AI-Powered Distillation（M3+）

### 9.1 Obsidian + Claude Code

```
M3+ 升级:
  · Obsidian + Smart Connections plugin
  · 连接 Claude / GPT API
  · 自动:
    · 找 atom connections
    · 建议 distill candidates
    · 生成 atom drafts from inbox

操作:
  1. 装 Obsidian + Smart Connections
  2. 配 OpenAI / Anthropic API key
  3. 周日 organize 时让 AI 第一轮 cluster (你审核)
  4. 周日 distill 时让 AI 生成 atom drafts (你 finalize)

时间从 30 min → 15 min/周
```

### 9.2 Karpathy LLM Wiki Pattern

```
[Karpathy 公开的 second brain 模板]

Folder structure:
  · 0-Inbox/         · 当周 capture (无分类)
  · 1-Resources/     · 持久 atoms (按 topic)
  · 2-Projects/      · 进行中工作
  · 3-Archive/       · 完成 / 归档

Naming convention:
  · YYYY-MM-DD-topic-quick-summary
  · 例: 2026-05-30-anthropic-32x-multiple-take

Backlinks:
  · 每个 atom 链接 2-3 个相关 atoms
  · 形成 knowledge graph
  · AI 引擎易解析
```

## 十二、5 件不要做

```
1. ❌ 试图 capture 所有看到的（5 min × 2 / 天 是上限）
2. ❌ 周中花时间 organize（只在周日做）
3. ❌ 用 second brain 做"工作纪录"（这是别的系统）
```
