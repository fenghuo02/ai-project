## 一、为什么必须跨平台

### 单 X 风险

```
风险 1: Foreign creator monetization 政策反复
        Nikita Bier 5/2026 提议过减权，Musk 推翻但风向已存。
        如果再启用，账号收入归零。

风险 2: 算法重大变动
        2026 年算法每 4 周月更。某次重大改版可能让你的格式不再受宠。

风险 3: 账号被误判 / 限流
        即使做对所有事，仍可能某天突然被算法误判。

风险 4: Foreign creator 可能限制 reach
        v3 已识别风险（Skub0k 是 GMT+8 watcher）。

风险 5: 单一收入来源
        revenue share + paid newsletter 都依赖 X 流量。
```

### 跨平台杠杆

```
1 篇 Sunday Long Read X Article
   ↓ 30 分钟
   Beehiiv newsletter 发出
   Substack Notes 重分发摘要
   LinkedIn Pulse 投资视角版本
   Bluesky long-form 镜像
   ↓
2 周内
   Substack Notes 流量 = 1×-3× X 流量
   LinkedIn Pulse = 不同受众池（投资 / 科技高管）
   Bluesky = AI / dev / fediverse 圈

1 条 Saturday vertical video
   ↓ 同时
   YouTube Shorts (revenue share)
   Instagram Reels
   TikTok
   小红书（中文圈备份）
   Bilibili（中文圈备份）
   ↓
观看池扩 5×，单一平台限流不致命。
```

### 投入回报

| 平台 | 复用文件 | 设置 1 次时间 | 周维护时间 | 预期月新增受众 |
|---|---|---|---|---|
| Beehiiv | newsletter md | 30 min | 0 min（自动）| 50-300 |
| Substack Notes | newsletter 摘要 | 20 min | 15 min | 30-150 |
| LinkedIn Pulse | newsletter 改写 | 30 min | 30 min | 20-100 |
| Bluesky | newsletter long-form 镜像 | 20 min | 10 min | 10-80 |
| YouTube Shorts | mp4 直传 | 15 min | 5 min | 50-500 |
| Instagram Reels | mp4 直传 | 15 min | 5 min | 30-200 |
| TikTok | mp4 直传 | 15 min | 5 min | 50-500 |
| 小红书 | mp4 直传 | 30 min | 15 min | 20-100 |
| Bilibili | mp4 直传 | 30 min | 15 min | 20-200 |

**总维护时间 ~90 分钟 / 周 = 跨平台总流量是 X 的 2-4×**。

## 二、平台分层

### Tier 1 · 主平台（X）

```
角色：核心生产 + 第一发布
内容形式：long-form 主推 / 5 series / 22 prompts
受众：全球 tech / markets / AI 圈
变现：revenue share（已限 Verified） + paid newsletter + 1:1 + Articles $1M Contest
启动节奏：M1 Day 1 已启动
维护：每天 30-45 min（v4 handbook §二 Daily SOP）
```

### Tier 2 · Newsletter 母船（Beehiiv 或 Mailchimp）

```
角色：邮箱沉淀 + 跨平台 lead 中心
启动：M1 Day 1 / 现在已启动 Tally form
迁移到 Beehiiv：M2 (100+ subs) 或 现在直接上 Beehiiv
为什么：
  · Boost network = newsletter 最大增长引擎
  · Recommendations 互推机制
  · X feed update 现在让 newsletter card 显示
  · Direct 邮箱沉淀，X 限流不致命
```

### Tier 3 · 长内容镜像（Bluesky · Substack Notes · LinkedIn）

```
角色：long-form 内容多平台分发
启动：M1 Week 2 起
为什么：
  · Bluesky 5/28 上线 long-form 对抗 X Articles
  · Substack Notes 是 Substack 圈的 X
  · LinkedIn Pulse 是投资 / 高管圈的 long-form 入口
```

### Tier 4 · 视频跨平台（YT Shorts / Reels / TikTok / 小红书 / Bilibili）

```
角色：vertical video 重分发 + 跨语言备份
启动：YT Shorts M1 Week 1（与 X video 同步），Reels / TikTok M2，小红书 / Bilibili M3
```

### Tier 5 · 备用 / 实验（Threads / Mastodon / Lemon8）

```
角色：备用入口
不主动维护，每月 1 次 cross-post 测试
```

## 三、Beehiiv Boost Network · 增长 #1 引擎

### 3.1 为什么 Beehiiv Boost 是必须

[2026 数据]：Beehiiv newsletters with active Boost campaigns grow 2× faster than non-participating ones（[beehiiv 案例](https://www.beehiiv.com/case-studies/the-small-bow)、[whop 2026 比较](https://whop.com/blog/beehiiv-vs-substack/)）。

Boost 机制：

```
Newsletter A 被 Newsletter B 推荐 → A 给 B 付 $1-5 / 个新订阅
                                    OR
Newsletter A & B 互推（双向，双方都拿到订阅，都不付钱）
```

### 3.2 启动 Beehiiv Boost 的 6 步

**Step 1**（M1 Week 2，约 30 分钟）：

1. 注册 Beehiiv（[beehiiv.com](https://beehiiv.com)，免费 plan 即可起步）
2. 创建 newsletter "Observer & Investor Weekly"（之前的 Musk Empire Decoder Weekly）
3. 设置 publication name / handle / logo（用 [visual/avatar.svg](./visual/avatar.svg)）

**Step 2**（30 分钟）：

4. 把 Issue 01-03 历史导入（之前 Tally 收的 + 截图发的内容重新格式化）
5. 设置 welcome email：第 1 封 = "thanks for subscribing + free Empire Cheat Sheet PDF link"

**Step 3**（30 分钟）：

6. 启用 **Boosts** 功能（在 Beehiiv 后台 Grow > Boosts）
7. 设置 incoming Boost 价格：$2 / sub（中等价位，过低自残，过高没人推）
8. 申请加入 outgoing Boost：选 5-10 个相邻 niche newsletter（tech / AI / markets / observer 类）

**Step 4**（5 分钟）：

9. 在 Beehiiv 设置 Recommendations 列表：3-5 个相邻 newsletter（不付费，纯互推）
10. 让别的 newsletter 看到你 = 他们订阅时也推荐你

**Step 5**（X 同步）：

11. X bio link 从 Tally form → Beehiiv subscribe page
12. Tally form 保留作为备用入口（为不愿留邮箱的人）

**Step 6**（持续）：

13. 每周日 newsletter 发布时，自动跑 Boost / Recommendations
14. 每月最后一天看 Boost 数据：哪些 boost 来源 ROI 最高 → 加大 / 减少投入

### 3.3 Boost 预算（M1-M5）

| 阶段 | 月 Boost 预算 | 预期新增 sub |
|---|---|---|
| M1（< 100 subs）| $0（先互推 + 看 Tally + 转 Beehiiv）| 20-50 |
| M2（100-500 subs）| $50-100 | 50-200 |
| M3（500-2K subs）| $100-300 | 200-800 |
| M4（2K-5K subs）| $300-800 | 800-2K |
| M5（5K-10K subs）| $800-1500 | 1500-3K |

**Boost ROI**：每个 sub 平均生命周期价值（LTV）= $20-50（按 paid 转化率 5-10%、$10/月、平均订阅 6 个月计算）。

如果你 Boost 一个 sub $2，预期 LTV $30 → ROI ≈ 15×。这是最高 ROI 增长投入之一。

### 3.4 Beehiiv X feed integration（v4 关键）

[2026 update](https://tahoor.beehiiv.com/p/x-feed-newsletter-update)：X 现在让 Beehiiv newsletter card 显示在 feed 中（card 比纯链接显眼）：

- 主推用 Beehiiv subscribe URL → X 自动 unfurl 为 card（不当成普通外链处理，曝光不减权）
- **这是 link-in-main 规则的少数例外**

实操：

```
主推（v4 链接规则一般禁止外链 → 但 Beehiiv card 例外）：
  empire decoder weekly · issue N just dropped.
  [一句话 take]

  [paste beehiiv URL → unfurl 为 card]

  bookmark this thread for the weekend.
```

注意：仅 Beehiiv URL 享受 card 待遇，**Substack / Mailchimp / 其他平台不一定享受**。每月最后一天验证一次。

## 四、Substack Notes · 长内容重分发

### 4.1 Notes 是什么

Substack 2024 推出的"X-style" 短推文功能。Substack 圈内置流量 = newsletter 受众。**Substack 用户**（写作圈、知识圈）跟 X 用户重合度 ~30-50%。

### 4.2 启动（M1 Week 2，30 分钟一次性）

```
1. 注册 Substack（[substack.com](https://substack.com)，免费）
2. 创建 publication（不直接 newsletter，主战场仍 Beehiiv，Substack 仅作为 Notes 入口）
3. 设置 bio + avatar（与 X 一致，用 visual/avatar.svg）
4. 启用 Notes
5. 在 Notes 加入 5-10 个相邻 newsletter 圈：
   · stratechery / not boring / lenny / not just AI / prag eng
6. follow + 偶尔 reply 他们的 Note
```

### 4.3 周节奏

| 周几 | Notes 内容 |
|---|---|
| Mon | Empire Map 摘要（4-6 句，不带 X 链接）|
| Wed | AI Race Card 数据（短表格）|
| Fri | Reading List 5 篇里**最有共鸣的 1 篇**单独发 Note |
| Sun | Long Read 1 段精华 + "full piece on substack [link]" |

**关键约束**：Notes **不要 cross-post X 主推 100% copy**。算法会判 spam，受众会觉得 lazy。每篇 Note 改写 30-50%，加 1-2 句"对 substack 圈来说"的额外背景。

### 4.4 Substack 长内容镜像策略

Substack 也是免费的 long-form 平台。但**不要**把它当主 newsletter 用：

- ❌ Substack 作为主 newsletter（10% 抽成 + Boost 不如 Beehiiv）
- ✅ Substack 作为 Bluesky / LinkedIn 的 cross-post 中心 + Notes 入口
- ✅ 每月 1-2 篇 Sunday Long Read 在 Substack 发**镜像版**（72 小时后，避免重复内容期）

## 五、LinkedIn Pulse · 投资 / 高管圈

### 5.1 为什么 LinkedIn

```
LinkedIn 用户画像：
  · 80% 知识工作者
  · 30%+ 是科技 / 投资 / 商业从业者
  · 投资圈在 LinkedIn 比在 X 更活跃（合规原因）
  · 高管 / VC / 招聘官在 LinkedIn 找内容

对应你的 v2 brand observer & investor：
  · Markets watch 板块 → LinkedIn 是天然受众
  · Empire tracking → LinkedIn 投资圈关心
  · Cultural observation → LinkedIn 是 audience-fit
```

### 5.2 启动（M1 Week 3，60 分钟一次性）

```
1. 完善 LinkedIn profile：
   · Headline："Independent observer · tech / markets / AI race"
   · About: observer-brand-pivot.md §四 起源故事 1-paragraph 版
   · Experience: 一栏 self-employed "Observer & Investor Newsletter" 起 2026-05
   · Featured: 1 链 Beehiiv subscribe page

2. Connect 100-200 个 first-degree relevant：
   · 之前认识的 + 业内 acquaintances
   · 不要狂加陌生 connection，质量 > 数量

3. Follow 30-50 个 thought leader：
   · Stratechery / Lenny / Not Boring / Tomasz Tunguz / VC partners
```

### 5.3 周节奏

| 周几 | LinkedIn 内容 |
|---|---|
| Mon 14:00 BJ（= 美东早上 2:00 = 不发，等周二）| skip |
| Tue 14:00 BJ（= 美东早 2:00 = 美工作日上午）| post 周一 Empire Map 改写版（去掉 X 段子味，加 1 句"investor angle"）|
| Thu 14:00 BJ | post 周三 AI Race Card 改写版 |
| Sun 23:00 BJ（= 美东 11 am 周日）| post Sunday Long Read 摘要 + "full piece on substack/x [link]" |

**关键约束**：

- LinkedIn 算法**不喜欢全小写 + lol / fwiw / ngl** ⚠️ 这与 v3 voice DNA 冲突
- LinkedIn 版本必须**首字母大写 + 完整标点 + 无 typo**（与 X 版本相反！）
- Voice 仍 dry / specific / observational / self-aware，但**正式化外壳**

### 5.4 Voice DNA · LinkedIn 适配版

LinkedIn 版本规则（与 X 版反向）：

```
✅ 首字母大写
✅ 完整标点符号
✅ 无 typo
✅ 较长段落（不要 X 的 1 句话格式）
✅ 偶尔 numbered list（LinkedIn 算法喜欢）

❌ "fwiw / ngl / tbh" 这种缩写
❌ 全小写
❌ Em-dash 不算红线（LinkedIn 接受）
❌ Emoji 仍少（≤ 1 / post）

仍保持：
✅ 数字密度 ≥ 3
✅ 招牌结尾（"3 takes. all could be wrong." 等）
✅ 不站政治 / 不给投资建议
```

### 5.5 Newsletter 在 LinkedIn

LinkedIn 内置 Newsletter 功能（不需要外链）：

- ✅ 启用 LinkedIn Newsletter
- ✅ 每月 1 次（**不**每周，会被 LinkedIn 算法判 spam）
- ✅ 内容 = 当月 Last-Day Monthly Map 改写版

## 六、Bluesky · Long-form 实验池

### 6.1 为什么 Bluesky

[2026-05-28](https://techcrunch.com/2026/05/28/bluesky-embraces-long-form-content-to-counter-x-articles)：Bluesky 推出 long-form support 对抗 X Articles。**新功能 + 早期红利窗口**。

```
Bluesky 用户画像：
  · ~50% 来自 X 不满者（tech 圈左派 + dev 圈）
  · ~30% AI / open source 圈
  · ~20% writer / journalist
  · 总用户 ~30M（远小于 X 的 600M），但**信噪比高**

对你的内容 fit：
  · AI race / Markets / Cultural observation 都 fit
  · Musk 主题在 Bluesky **不太友好**（受众反 Musk 居多）
  · Empire tracking 可以发，但要更中性 / 数据驱动
```

### 6.2 启动（M2 Week 1，60 分钟一次性）

```
1. 注册 Bluesky（[bsky.app](https://bsky.app)，免费）
2. 用 same handle "@Skub0k.bsky.social" 或 custom domain
3. 完善 profile（同 X，但 bio 调一个版本）：
   "an observer & investor.
    decoding tech, markets, IPOs, AI, and the empires being built right now.
    🌏 perspective from outside silicon valley.
    free monthly tracker pdf below."
4. Follow 30-50 个相关 dev / AI / tech writer
```

### 6.3 周节奏

| 频率 | 内容 |
|---|---|
| 每周 1 长内容 | 周日 Long Read 镜像版（用 standard.site / Bluesky native long-form） |
| 每周 1-2 短 post | 当周最有共鸣的 take（不要 cross-post X，原创再写） |
| 不发 | Musk-only 内容（Bluesky 受众不友好）|

### 6.4 关键约束

- ❌ 不要在 Bluesky 发 Musk 帝国捧杀（受众会反感）
- ✅ 中性视角的 Empire tracking 仍可发（数据驱动）
- ✅ AI race / Frontier tech / Markets observation 友好
- ✅ Voice DNA 与 X 一致（dry / deadpan / specific）

### 6.5 Bluesky vs Substack vs LinkedIn 投入优先级

```
M1-M2: Beehiiv（必须）+ Substack Notes（必须）+ LinkedIn（30 min/周）
M3+:   加 Bluesky long-form 镜像（30 min/周）
M4+:   加 Mastodon / Threads（30 min/周，备用）
```

## 七、视频跨平台分发（已部分在 vertical-video-pillar.md）

### 7.1 同 mp4 多平台 SOP

每周六 11:00 BJ 渲染完 mp4 后：

| 平台 | 上传时间 | 文案调整 |
|---|---|---|
| **X**（主） | 11:30 BJ | 招牌 hook + 招牌结尾 |
| **YouTube Shorts** | 11:35 BJ | Title 60 字符内 + caption 同 X |
| **Instagram Reels** | 11:40 BJ | Caption 加 5-10 个 hashtag（IG 算法仍重 hashtag）|
| **TikTok** | 11:45 BJ | Caption 加 5-8 个 hashtag + "for you" 类钩子 |
| **小红书** | 13:00 BJ（错峰）| 标题转中文 + 中文 caption（**翻译 + 适配**，不是直接 google translate） |
| **Bilibili** | 13:10 BJ | 标题转中文 + 短 description |

### 7.2 Cross-post 时间错峰原因

X / YT Shorts / IG / TikTok 的算法都"读"内容。完全同时发会被判定为"批量分发" → 跨平台都减权。错峰 5-10 分钟规避。

### 7.3 中文圈备份策略

小红书 / Bilibili 是 **观察备份**，不是主战场：

- 起步阶段 ROI 低（不到 X / YT 的 1/10）
- 但抓住 1 个爆款可以带 1-5K 中文圈关注（高 LTV，未来变现潜力）
- 策略：**只发表现最好的视频**（Saturday video M2 起每月选 ROI 最高 1 条同步）

│  · LinkedIn Pulse                    │

└──────────────────────────────────────┘
                ↓
┌──────────────────────────────────────┐
│  关注层                              │
│  · X follower（最大池）              │
│  · YouTube subscriber                │
│  · LinkedIn connection               │
│  · Substack follower                 │
└──────────────────────────────────────┘
                ↓
┌──────────────────────────────────────┐
│  Newsletter Sub（核心 lead）⭐       │
│  · Beehiiv free                      │

                ↓

**所有平台都指向 Newsletter Sub**。这是漏斗最稳定的层，X 算法变动 / 平台限流都不影响。

## 九、平台失败 / 退场标准

```
对每个平台 (Tier 3-5)，连续 3 个月：

  · 月新增 follower < 30
  · CTR 到 newsletter < 1%
  · 单平台维护时间 > 30 min/周
  → 暂停 / 仅 1×/月简发

永远不停 X (Tier 1) 和 Beehiiv (Tier 2)。
LinkedIn / Substack 退场前先减频（每周 1 次 → 每月 2 次）。
小红书 / Bilibili 起步 ROI 必然低，**M3 之前不评估**。
```
