## 一、为什么需要重做 revenue 设计

### 1.1 v3 的局限

```
v3 收入路径（按 M2-M5）:
  · X ad share              主动
  · Beehiiv paid newsletter 主动
  · 1:1 Briefing            主动（极重）
  · Empire Tracker Pro PDF  半主动
  · Annual Outlook PDF      半主动
  · Sponsorship             主动 + 不可控
  · X Articles $1M Contest  半主动（v4 加）

总 = 100% 主动 / 半主动
零 passive income
```

**问题**：

- 你休息 1 周 = 收入掉 50%（无 passive 缓冲）
- Burnout 时 = 0 收入
- 跨平台扩张 = 工作量翻倍但收入没多平台对应增加

### 1.2 v5 全栈设计

```
v5 收入分层（M5 时目标）:

  Active 50%
    · Beehiiv paid newsletter
    · 1:1 Briefing
    · Sponsorship
    · X Articles Contest

  Semi-passive 30%
    · Empire Tracker Pro PDF (季度更新)
    · Annual Outlook PDF (年度)
    · Saturday Video YT Shorts revenue
    · X ad share

  Passive 20% ← 新增
    · Affiliate / referral 30+
    · Custom GPT subscription (M5+)
    · Course / mini-product (M5+)
    · 投资收益 (treasury / personal portfolio)
```

收入分散度目标：

```
M2: 90% 主动 / 10% semi-passive
M3: 80% / 15% / 5%
M4: 70% / 20% / 10%
M5: 50% / 30% / 20% ← v5 目标
M6+: 40% / 30% / 30%
```

### 1.3 行业证据（2026-05 联网验证）

- [Creator Revenue Stack 2026](https://creator-revenue-stack.netlify.app/)：30+ vetted affiliate up to $1500/referral
- [Whop affiliate guide](https://whop.com/blog/affiliate-marketing-programs/)：30% lifetime commission models
- [JourneyBee 2026](https://journeybee.io/resources/top-100-referral-programs-to-make-money)：$2000 bounties + recurring SaaS commissions
- [Automateed Diversify Income 2026](https://www.automateed.com/how-to-diversify-income-as-a-creator)：community-led models + smarter automation = scaling

  · Mercury bank                    已经用
  · Wise                            已经用
  · TweetHunter (后续开)             5 min
```

### 4.2 Tier 2 (M3 启动)

每月加 2-3 个 affiliate，3-6 个月完成 Tier 2 全部。

### 4.3 Affiliate 推广纪律

```
✅ 只推自己真实在用的
✅ 在 Reading List / Sunday Long Read / 1:1 / Newsletter 中**自然提及**
✅ 在 bio / pinned / footer 加 "tools i use" 链接合集
✅ 月度 review 哪些 ROI 最高加大投入

❌ 在主推 / Quote-tweet 直接发 affiliate link (X 严打)
❌ Affiliate spam (mass DM / mass mention)
❌ 推自己没用过的 (信誉崩)
❌ 隐藏 affiliate 关系 (FTC 违规)
❌ Reply guy 战场推 affiliate (违反 collab 协议)
```

### 4.4 Affiliate disclosure (FTC 合规)

```
Beehiiv newsletter 末尾必须 (在所有 issue):
"some links above are affiliate. i use these tools daily.
 your support helps keep this newsletter free."

X bio:
"tools (some affiliate): [link]"

详见 [legal-compliance.md §五 FTC Disclosure](./legal-compliance.md)
```

```
名: "Empire Decoder GPT"
描述: musk empire / AI race / IPO 实时观察 + decoder dictionary
功能:
  · Decode any musk tweet

  · Free tier: 5 queries / day

### 5.2 上架步骤

```
1. ChatGPT 创建 GPT (1 hr)
   · Instructions = compress 你的 voice DNA + brand v2
   · Knowledge base = 上传所有 long read articles + cheat sheet PDF + decoder dictionary

2. 测试 100 次 query (5 hr)
   · 各类 prompt 测
   · 修 instruction 直到 90%+ 满意

3. GPT Store 上架 (30 min)
   · Title + description
   · Cover image (用 visual/template-decoder-frame.html)
   · 价格设置

4. Marketing (持续):
   · 1 篇 Sunday Long Read 介绍这个 GPT
   · Bio 加链接
   · Newsletter 推荐
   · Reading List 偶尔提
```

## 九、Revenue 预防

### 9.1 单一来源风险

```
任何单一 revenue stream > 60% 总收入 = 预警
任何单一 stream > 80% = red alert

应对：
  · 立刻加大 diversification 投入
  · 减少对该 stream 的依赖
  · M5 时无任何单一 stream > 40% 才健康
```

### 9.2 Burnout 风险

```
1:1 Briefing > 20 单 / 月 = 预警
1:1 + Sponsorship 主动工时 > 20 hr / 月 = red alert

应对：
  · 涨价 (减需求量保收入)
  · 增加 backlog 库存
  · 加快 VA / contractor
```

### 9.3 平台风险

```
单一平台 (X / Beehiiv) > 70% 总收入 = 预警

应对：
  · 加大 [cross-platform-funnel.md](./cross-platform-funnel.md)
  · 多平台分散
  · M5 时 X 占比应 ≤ 50%
```

## 十、长期 mindset

### 10.1 5 件不要做

```
1. ❌ 假设 1:1 / Sponsorship 可以无限规模 (天花板低)
2. ❌ 太早启动 paid (M2 之前 audience 不值得 paid)

4. ❌ Founding Member 太早开 (受众不到 30K = 没足够 demand)
5. ❌ 拒绝 passive income (拒绝时间杠杆 = 拒绝 big-V)

### 10.2 5 件要做

```
1. ✅ M2 立刻接 Tier 1 affiliate (低成本，高 ROI)
2. ✅ M3 测试 1:1 价格 ($99 → $199 → $299) 看 demand 弹性
