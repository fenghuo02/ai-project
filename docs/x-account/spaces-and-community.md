## 一、为什么需要这一层

### 流量号 vs 留存号

| 维度 | 流量号 | 留存号（大 V）|
|---|---|---|
| 受众关系 | 单向广播 | 双向社群 |
| Churn 率 | 高（涨粉等于失粉）| 低（受众有"跳出成本"）|
| 货币化天花板 | 30% （只能 ad share + 偶尔 sponsorship）| 5-10x（订阅 / 课程 / 1:1 / 社区会员）|
| 危机抗性 | 一次限流 = 灾难 | 受众通过 Spaces / Community 找到你 |

### 三个层级的留存机制

```
┌──────────────────────────────────────────────────┐
│ Tier 0 · 公开内容（所有 follower）               │
│   · daily 7 posts                                │
│   · 5 signature series                           │
│   · cheat sheet pdf                              │
│   · free newsletter                              │
└──────────────────────────────────────────────────┘
                ↓ (subset 转化)
┌──────────────────────────────────────────────────┐
│ Tier 1 · Friday Spaces（free, 全开放）           │
│   · 每周 30 分钟                                 │
│   · open Q&A 后段                                │
│   · 录音放 newsletter / X Article 24h 内         │
│   · M2 触发条件：1K+ followers                   │
└──────────────────────────────────────────────────┘
                ↓ (深度 subset)
┌──────────────────────────────────────────────────┐
│ Tier 2 · X Community（free, gate-keeper）        │
│   · gate: 关注 30 天 + 至少 1 次互动             │
│   · 用途：discord-like 长讨论                    │
│   · M3 触发条件：5K+ followers                   │
└──────────────────────────────────────────────────┘
                ↓ (核心 subset)
┌──────────────────────────────────────────────────┐
│ Tier 3 · Paid Newsletter $10/月                  │
│   · 结构性权益（不只是"更频"）                   │
│   · 月度 group AMA                               │
│   · Reading List 深度版（10 instead of 5）       │
│   · Cheat sheet 提前 3 天                        │
│   · 1:1 booking 8 折                             │
│   · M2 触发条件：30+ DM 询问 paid 内容           │
└──────────────────────────────────────────────────┘
```

> 关键：**不能反向跨级**。Tier 2 / 3 一定从 Tier 1 升级而来，不能直接面向陌生人推。

## 二、Friday Spaces · 周五傍晚 30 分钟

### 启动条件

**满足 2 个 + 才启动**：

- [ ] Followers ≥ 1,000
- [ ] 至少 30 天内有 2 条主推 impression > 10K
- [ ] 至少 5 个 5K+ 粉的人在主动 reply
- [ ] 个人有 30 分钟连续表达英文的能力（不卡顿）

不满足 = **不要硬开**。一个 0 听众的 Space 比不开 Space 伤害大（X 算法会标记你"低质量内容创作者"）。

### 时段固定

```
每周五 21:00-21:30 北京 · Beijing
       = 每周五 09:00-09:30 美东 · 通勤 / 早咖啡时段
       = 每周五 06:00-06:30 美西 · 起床刷推时段
```

不浮动。一旦受众建立期待，**任何一周漏掉都伤害 trust**。

### 30 分钟标准 agenda

```
21:00 - 21:03 (3 min)
  · 入场 buffer：欢迎 + 自我介绍（"i'm @Skub0k. observer & investor.
    we'll talk about [当周主题] for the next 30 min."）
  · 念 housekeeping：录音 ON / 周日发到 newsletter
  · "请把麦克风申请放在 raised hands"

21:03 - 21:18 (15 min) · 主讲段
  · 当周主题 5-7 个 talking points
  · 每个 point 1-2 分钟
  · 允许自己 stutter / pause / "let me think"
    （这恰恰是 anti-AI tone 的 Spaces 版）

21:18 - 21:28 (10 min) · Q&A 段
  · 接 3-4 个听众问题
  · 每个回答 ≤ 2 分钟
  · 1 个回答里至少 1 次"i don't know" 或 "i'm not sure"

21:28 - 21:30 (2 min) · 收尾
  · "next friday: [下周主题]"
  · "newsletter sub link in pinned tweet"
  · "录音 24h 内发出"
  · END
```

**关键纪律**：30 分钟 = 30 分钟。**31 分都不行**。受众的"跳出成本"是低估的，准时结束 = 受众下周还来。

### 主题轮换（4 周 1 个循环）

| 周 | 主题 |
|---|---|
| Week N · 1st Fri | 当周 Empire Map 的扩展讨论（接周一 thread）|
| Week N · 2nd Fri | 当周 AI Race Card 数据深挖（接周三 card）|
| Week N · 3rd Fri | 当周 Reading List 5 篇之一深读（接周五 list）|
| Week N · 4th Fri | open mic · 听众提议主题或当周突发新闻 |

第 4 周 open mic 故意留空，应对 IPO / 财报 / 重大事件。

### 工具

- **Host 端**：X Premium+ 内置 Spaces hosting（无费用）
- **录音保存**：Spaces 自动保留 30 天 → 第 24h 用 Spaces clipper 截 5 分钟精华片段重发
- **辅助工具**：手机 + 一对耳机就够了。**不要**搭复杂的混音设备
- **网络**：必须有线网或稳定 4G/5G。WiFi 抖一次 = 全场断（X 不会自动恢复）

### 反 AI 味（Spaces 版）

X Spaces 是音频形式，AI 味的危险点不一样：

| AI 味 | 真实人 |
|---|---|
| "Today we'll explore three key dimensions of..."  | "ok so / let me start with / the thing that bugged me this week..." |
| "This is a fascinating topic that..."  | "this got weird, here's what i mean..." |
| 完美的 transitions（"Furthermore", "Moreover"）| 真实的 "uh / so / ok / wait" |
| 每个回答都 3 个 bullets | 1 句直接答 |
| 紧张地读稿 | 看一眼 prep notes 但不读 |
| 每个问题先 "Great question!" | 直接回答 |

### 7 个常见 Spaces 危机 + 应对

#### A · 0 个听众进来（启动后 3 分钟）

- **不要**马上结束
- 用第 1 个 5 分钟讲完 1 个完整 talking point（即使没人听，录音会被周日 newsletter 发出去）
- 然后说"i'm going to keep this brief today, sticking to 1 point" → 13 分钟提早结束
- 数据：进了 0 人但录音发出去 → 后续 24h 仍能从 newsletter 拿 100-300 听众

#### B · 听众进来后 5 分钟掉到 0

- 不是你的问题，是 X Spaces 的 push 通知失败
- 继续讲完，**不要**改话题取悦
- 录音照常发

#### C · 听众主动挑衅 / hostile question

- 让他说完
- 一句话同意可同意的部分（"ok, the part about [X] is fair"）
- 一句话清晰反对的部分（"the part about [Y] i actually disagree because [一个数字]"）
- 接下一个 raised hand
- **不要**进入辩论模式（这是 Q&A，不是 debate）

#### D · 听众讲了 5 分钟不停

- 等他停顿
- 接一句"thanks. quick reaction:"（强行截断但礼貌）
- 30 秒内回应
- 接下一个

#### E · Co-host / 邀请的人不来

- 单飞继续。**不要**等 / 不要把 Space 暂停
- 周日发录音时不提及缺席
- 私下 follow up（不公开追究）

#### F · 自己讲飞了，超过 30 分钟

- 在 31 分时强行收尾："we're at the bell, see you next friday"
- **绝对不**说"this got long, let me wrap up"（30 分纪律破一次受众下周就少 20%）

#### G · 网断了 / app crash

- 重新进入 Space（X 会保留 host 30 秒重连窗口）
- 简单一句"sorry, dropped. continuing where we were"
- 不要重新自我介绍（破坏节奏）

### Spaces 之后 24h SOP

| 时间 | 动作 |
|---|---|
| Fri 21:30 | Space 结束 |
| Fri 21:35 | 立刻发一条 follow-up 推：录音 link + 3 个 takeaway bullets |
| Fri 22:00 | 把 Space 加进当晚 newsletter prep 工作（Sunday newsletter 引用）|
| Sat 任意 | 用 Spaces clipper 截 1-2 个 1-2 分钟金句 → 重发到 timeline |
| Sun 21:30 | newsletter 发 + 嵌入 Spaces 录音 link |

## 三、X Community · 关注者私聊圈

### 启动条件

**满足全部才启动**：

- [ ] Followers ≥ 5,000
- [ ] Friday Spaces 已稳定跑 8 周
- [ ] 至少有 100 个真实"反复参与 Spaces"的 ID（不是 bot）
- [ ] 个人有意愿每周 30 分钟看 Community feed

### 设计

```
Community 名: Empire Decoder Lounge
Description: independent observers reading the same filings.
             tech / markets / AI / cultural observation.
             discussion-grade, not headline-grade.
```

### 加入门槛（防 bot + 提升信号噪音比）

```
□ 必须关注 @Skub0k 30 天 +
□ 至少 1 次主推 reply / Spaces raised hand
□ Bio 不能有 "crypto / signal / pump / SMM"
□ 申请时一句话答："what's one thing you read this week worth bookmarking?"
```

每周 host 手动审 5-10 个新申请。**不要**全自动通过（社区死亡的开始）。

### Posting rules（贴在 Community 描述里）

```
1. no politics / partisan content
2. no financial advice
3. no self-promo（除了 weekly intro thread）
4. quality > quantity（一周不强求发，重要 > 频繁）
5. disagree freely · with sources
```

### Host 节奏

| 频率 | 动作 |
|---|---|
| 每周 1 次 | 发 1 条 community-only 帖（Reading List 深度版 / Friday Spaces 内幕 / 不公开数据）|
| 每月 1 次 | host 1 次 community-only Space（30 min, members only） |
| 每季度 1 次 | "what should observer cover next quarter" 调研 thread |

### Community 不当 paid 用

X Community 在 2026 年仍是 free 功能。**不要**把 Tier 2（Community）和 Tier 3（Paid Newsletter）混淆：

- Tier 2 = free，gate 是"参与质量"
- Tier 3 = paid，gate 是"$10/月"

混在一起 → 受众不知道为什么要付费 → 转化率归零。

## 四、Paid Newsletter $10/月 · 结构性会员权益

### 现状问题

旧设计（[x-strategy.md §四](./x-strategy.md)）：

```
$10/月 = "更频"（每日早报 + 周度数据表）
```

**问题**：受众找"频率"可以去 ChatGPT 自己生成。"更频"不是付费理由。

### 新设计 · 结构性权益

| # | 权益 | 受众感知 |
|---|---|---|
| 1 | **月度 group AMA**（30 min Zoom / X Spaces members-only）| "我能直接问 host" |
| 2 | **Reading List 深度版**（10 篇 instead of 5，含 paywall 后内容摘要）| "看到别人没看到的" |
| 3 | **Cheat Sheet PDF 提前 3 天**（周四发，免费版周日发）| "我先于市场" |
| 4 | **Annual Outlook 提前一份**（M5 时 $99/年产品）| "省 $99 + 先看到" |
| 5 | **1:1 Briefing 8 折券**（$99 → $79，无限次）| "降价券永远在" |
| 6 | **Article paywall 内容**（M3+ 起每月 1-2 篇付费）| "深度 + 完整数据" |
| 7 | **Empire decoder lounge 内核区** "premium thread"（X DM group 形式，host 答疑） | "更近的距离" |

### 价格结构

```
M2 launch (5K+ followers)
  Free                 $0      monthly tracker pdf + weekly newsletter
  Paid Member          $10/月  上面 7 项权益
  Paid Member 年付     $99/年  = $8.25/月（17% off）
  Founding Member      $499/年 含 4 次 1:1 + lifetime member badge
                              （上限 50 个，建立 inner circle）
```

### Founding Member 池子的特殊价值

50 个 founding member 不是收入的核心，是**社区核心层**：

- 永久会员 badge（X profile 显示）
- 每次有重大决策（栏目改 / 视觉改 / 价格改）先问他们
- 他们在 X 公开发言时 = 自然广告

50 个 × $499 = $25K 一次性收入，但**这 50 人是 follow-up 5 年的核心**。值得花精力筛。

### 货币化护栏

- ❌ 不要给 paid 用户更"激进"的 take（risk 偏移）
- ❌ 不要给 paid 用户"内幕消息"（违反信任 + 合规风险）
- ❌ 不要把 free 用户的频率降到看起来"被薅"的程度
- ❌ 不要在 X 主推说"this thread is for paid only"（伤害免费层）
- ✅ Paid 内容是 **深度 / 提前 / 工具**，不是"更好的视角"

## 六、第一次 Spaces 启动 checklist

只在 §二启动条件全部满足后做：

- [ ] 决定首次主题（建议：当周 Empire Map 的 Q&A 延伸）
- [ ] Spaces 标题（≤ 50 字符）："empire map week N · open Q&A"
- [ ] 30 分钟 agenda 草稿（手写 5-7 个 talking points + 时间分配）
- [ ] 周三在 X 公告"this friday 21:00 BJ"
- [ ] 周五早 09:00 二次提醒
- [ ] 周五 20:30 准备：手机充电 + 网络测试 + 耳机检查
- [ ] 周五 20:55 提早 5 分钟开 Space（让早到的人先进）
- [ ] 21:00 准时讲第一句
- [ ] 21:30 准时收尾
- [ ] 21:35 发录音回顾推
- [ ] 22:00 加进当晚 newsletter prep
- [ ] 周日 newsletter 嵌入录音 link

## 七、Spaces / Community 失败 / 退场标准

### Spaces 失败标准（连续 4 周）

| 指标 | 失败阈值 | 动作 |
|---|---|---|
| Live 听众数 | < 5 | 改时段 / 改主题 |
| 录音 24h 收听数 | < 30 | 改 promo 节奏 |
| Spaces 后 1 周 follower | 净流出 | 复盘内容 + 改 |
| 准备时间 | > 60 min | 改 agenda 模板更省力 |

**永远不停 Friday Spaces 这个槽**。只能 down-shift（30min → 15min）或 reframe（独白 → 邀请 1 co-host）。

### Community 失败标准（连续 8 周）

| 指标 | 失败阈值 | 动作 |
|---|---|---|
| 月度新增成员 | < 5 | 暂停审批 1 周，关注内部活跃度 |
| 月度成员发帖数 | < 0.5 / 人 | reframe 为 host-led（host 起话题）|
| Member churn 率 | > 10% / 月 | 改 posting rules |

如果 16 周后还是不健康 → **停 Community，集中精力 Spaces + Paid**。Community 是"3 件套里最可选的一个"。

## 十、Bot / 假账号防御

X Spaces / Community 是 bot 重灾区。每月做 3 件事防御：

1. 看 Community 新申请的 bio 和发推历史 → 任一可疑特征拒绝
2. Spaces 看 raised hands 的 follower 池子 → 大量 follower 0 / 1 = 可能是 bot 池子
3. 每月一次 export Community member 名单 → 用 Botometer 类工具批量过 5%
