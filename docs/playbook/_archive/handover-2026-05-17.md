## 5 · 八字 → 命书颜色映射规则（PR #33 引入）

模板 v1.0 起，命书 PDF 的封面 + 卷扉 + 装饰元素**全部由 yaml 的 `colors.primary[0].hex` 驱动**。

写新客户 yaml 时**必须按八字推导**填，不可复制粘贴：

| 客户日主 | 月令季节 | 推荐主色 | hex |
|---|---|---|---|
| 甲乙木 / 春生 | 春 | 苍翠 / 嫩绿 | `#2D5F3F` / `#8FB573` |
| 丙丁火 / 夏生 | 夏 | 朱砂 / 朱橘 | `#C0392B` / `#E67E22` |
| 戊己土 / 季月 | 任意 | 暖驼 / 烛光金 | `#C9A66B` / `#D4AF37` |
| 庚辛金 / 秋生 | 秋 | 月白 / 月光银 | `#FAF7F2` / `#E8E6E1` |
| 壬癸水 / 冬生 | 冬 | 墨青 / 雾蓝 | `#2C3E50` / `#5D7B91` |

> ⚠️ 这只是**起点参考**，实际推导要看：
> 1. 日主 vs 月令的强弱
> 2. 调候/扶抑/通关用神
> 3. 喜神 / 忌神
>
> 例：壬水日主生于秋月（金生水）反而强，需要的是**疏泄**而非补水——
> 主色应该选暖驼/烛光金（耗水），不是更多墨青。
> 主理人本人就是这个例子。

## 6 · 沙盒环境的坑（新 AI 必须知道）

| 坑 | 表现 | 解决 |
|---|---|---|
| 沙盒装不了 PDF 工具 | `pip install weasyprint` 走代理 403 | 用 GitHub Actions CI 出 PDF（artifact）|
| `dnf install` 也 403 | Amazon Linux 仓库被代理拦 | 同上，必须依赖 CI |
| `fs_write` 中文路径 bug | 写到 `/projects/sandbox/` 而非 `/projects/sandbox/ai-project/` | 写完用 `cp` 或 `mv` 移过来再校验 |
| YAML 中文引号嵌套 | 半角双引号嵌套半角双引号会语法错 | 内嵌引号用全角「」（U+300C/D） |
| `run_command` 工具偶尔不可用 | "Tool not available" | 改用 `execute_bash`（必须传 `cwd` 参数）|
| 校验中文目录名 | python3 默认 LANG=POSIX 找不到中文目录 | 加前缀 `LANG=C.utf8 LC_ALL=C.utf8 PYTHONUTF8=1` |
| 默认 python3 缺 yaml | `ModuleNotFoundError: yaml` | 用 `/opt/amazon/bin/python3.11`（自带 pyyaml）|
| pull 工具有时报 divergent | 本地有未提交改动时 | 先 `git reset --hard` 再 pull |

### 校验客户 yaml 的标准命令

```bash
LANG=C.utf8 LC_ALL=C.utf8 PYTHONUTF8=1 \
  /opt/amazon/bin/python3.11 mingshu-builder/build.py --validate W女士-1986
```
