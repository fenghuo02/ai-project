# 五行肖像 · AI 背景替换 SOP

> **核心原则**：客户面部 + 身体 100% 保留实拍 · AI 仅替换背景 · 全程按八字推命局定制配色
> **法律性质**：常规商业修图，不是深度合成。但仍需 AIGC 标识 + 客户授权。

---

## 一、入口规则（强约束）

### 我（Kiro）能接受的输入

| ✅ 必须 | 说明 |
|---|---|
| 客户**真实拍摄**照片（米白/浅灰底最佳） | 不能是网图 / 艺人照 / 朋友照 |
| 客户**生辰八字** | 年月日时四柱完整，否则推不出用神 |
| 客户**已签拍摄授权**（含 AI 后期修图条款）| 标准合同的 Clause 3.2 |

### 我（Kiro）会拒绝的输入

| ❌ 拒绝 | 原因 |
|---|---|
| 网图 / 名人照 / 艺人照 | 肖像权风险（业务计划 §11.4）|
| 没有生辰八字的肖像生成 | 配色无依据，等于普通修图 |
| 要求"基于客户照换脸到不同场景"| 这是 deepfake，不是背景替换 |
| 要求把客户原片提交进 GitHub 公开仓库 | PIPL + 肖像权双红线 |

---

## 二、按八字定制的 SOP（5 步）

### Step 1 · 命理推导（我做 / 5 分钟）

输入：年月日时八字
输出：**用神配色 + 最佳门 + 类型一句**

例（W 女士 1986 丙寅丁酉己未壬申）：

```
日主：己土
月令：酉（仲秋金旺）
强弱：偏弱
用神：丙火 / 丁火（年月双透）
喜神：戊土
忌神：金
本命主色：朱砂 #C0392B
本命辅色：朱橘 #E67E22
辅助根色：暖驼 #C9A66B
避忌色：月光银 #E8E6E1
最佳门：景门（灵感表达）
类型一句：秋土待火 · 金叶满径
```

### Step 2 · 写命书 yaml（我做 / 30 分钟）

按 `mingshu-builder/clients/_template/data.yaml` 模板填，重点填三处：

| 字段 | 内容 |
|---|---|
| `colors.primary` | **决定整本命书 PDF 封面颜色** ⭐ |
| `colors.forbidden` | 客户面要避开的色 |
| `ba_men[*].is_best` | 选 1 门为客户最佳匹配（含专属彩蛋）|

> 🚨 **关键** ：colors.primary[0].hex 会**驱动整本命书 PDF 的视觉主调**——封面/卷扉/章节标题/装饰线全部按这个色变。
>
> - W 女士 → 朱砂封面 ✅
> - 张先生（壬水日主，喜暖驼）→ 暖驼封面 ✅
> - 李小姐（庚金日主，喜墨青）→ 墨青封面 ✅

### Step 3 · 写五行背景 prompt（我做 / 20 分钟）

按客户主色 + 五行配比，给出 5 张五行背景 AI prompt（**无人物**）。

#### 万能母版

```
新中式东方美学风格，水墨意境，宋画工笔感，
留白充足，构图克制，柔和宣纸质感，
矿物颜料的颗粒感，低饱和度，工笔写实，
日系治愈风 + 国风手作感，
4K 高清，3:4 竖版构图，
画面中央留出 60% 空间放人物（不出现任何人物）
```

#### 五行背景模板（按客户主色调整）

> 关键：**主色按客户实际推导的本命主色填入**，不是默认值

##### 火 · 朱砂背景

```
【母版风格】+
室内场景，朱漆大门特写，门上贴金色福字，
木质门框，门前一对小红灯笼半隐入画面边缘，
冬日清晨的暖阳从画面右上方斜射，
朱砂 {{客户主色 hex}} + 烛光金 {{辅色 hex}} 主调
```

##### 木 · 苍翠背景

```
【母版风格】+
半透茶席背景，远处高大竹林散景柔焦，
桌上素面紫砂壶 + 一卷线装古籍，
午后柔光从左前方洒下，
苍翠 {{主色}} + 暖驼 {{辅色}} 主调
```

##### 土 · 暖驼背景

```
【母版风格】+
室外秋日傍晚的麦田，麦穗被夕阳染成金黄驼色，
远处淡淡的山影，地平线在画面下三分之一处，
夕阳从画面左上方斜射，
暖驼 {{主色}} + 烛光金 {{辅色}} + 赭石 主调
```

##### 水 · 墨青背景

```
【母版风格】+
书房红木桌前的窗景，窗外夜色 + 一轮淡月，
桌上一盏暖光台灯 + 一杯热茶，
窗内暖光与窗外冷夜形成对比，
墨青 {{主色}} + 烛光金 {{辅色}} + 雾蓝 主调
```

##### 金 · 月白背景

```
【母版风格】+
工作室落地窗自然光下，留白米白宣纸墙，
左侧浅银色装饰花瓶虚化，
柔和侧逆光从右侧窗外洒入，
米白 {{主色}} + 月光银 {{辅色}} + 烛光金点缀 主调
```

> ⚠️ **客户主色已在 yaml 里**——我（Kiro）每次给具体客户写 prompt 时，会**主动把 `{{客户主色}}` 替换成 ta 的实际 HEX**，比如：
> - W 女士火背景：朱砂 #C0392B + 烛光金 #D4AF37
> - 主理人土背景：暖驼 #C9A66B + 烛光金 #D4AF37

### Step 4 · 主理人执行（你做 / 90 秒/张 × 5 张）

#### 在即梦 / 通义万相里：

```
1. 上传客户原片 → 点 "AI 抠图" → 保存透明底人像 PNG  (10s)
2. 切换到 "文生图" → 粘贴上面 5 张背景 prompt 之一    (40s)
3. 切换到 "图层合成" → 上传抠出的人像 PNG          (20s)
4. 调整人像缩放和位置 → 导出 PNG                  (10s)
5. 美图秀秀 / Photoshop → 右下加 AIGC 水印 ≥10pt 半透白  (10s)
```

**5 张图 = 7-8 分钟搞定**。

#### 文件命名规则（重要！）

按 `mingshu-builder/clients/<id>/images/` 命名：

```
images/
├── portrait.png       (封面用 · 默认用"土"或客户喜欢的那张)
├── jing.png           (景门 · W 女士最佳门)
├── kai.png            (开门)
├── xiu.png            (休门)
├── sheng.png          (生门)
├── shang.png          (伤门)
├── du.png             (杜门)
├── si_men.png         (死门 → 客户面叫"断舍释怀")
└── jing_men.png       (惊门 → 客户面叫"清明心安")
```

build.py 会自动按 key 从 images/ 找图。找不到的就用 CSS 渐变 fallback。

### Step 5 · 跑 build 自动出 PDF（CI 做 / 3 分钟）

```bash
# 本地（如果有 Python + weasyprint）
cd mingshu-builder
python3 build.py W女士-1986
open clients/W女士-1986/output.pdf

# Windows 用户：直接 push 触发 CI（推荐）
git add mingshu-builder/clients/W女士-1986/
git commit -m "feat: W 女士命书图片就绪"
git push
# 等 3 分钟 CI 跑完 → 在 Actions 页面下载 ci-built-pdfs 工件
```

#### PDF 视觉自动按主色变 ⭐

| 客户 | 主色 | PDF 封面 | 卷标题 | 装饰线 |
|---|---|---|---|---|
| W 女士（己土秋金）| 朱砂 #C0392B | 朱砂渐变 | 朱砂大字 | 朱砂虚线 |
| 主理人（壬水秋金）| 暖驼 #C9A66B | 暖驼渐变 | 暖驼大字 | 暖驼虚线 |

## 三、合规护栏

### 3.1 AIGC 标识（强制）

每张五行肖像**必须**在右下角加水印：

```
本作品视觉部分由 AI 辅助创作 · AIGC
```

字号 ≥10pt，半透明白底黑字（深色背景反之），不可擦除。

### 3.2 客户授权（强制）

拍摄前签 `docs/contracts/c-end-portrait.md`（待 PR），包含：
- 拍摄肖像授权
- AI 后期修图（背景替换）授权
- 案例展示授权（脱敏到首字+生年）

### 3.3 禁止 deepfake（红线）

**永远不**做：
- 把客户照片输入 face-swap 模型
- 用 ControlNet OpenPose 让客户"做出 ta 没做过的姿势"
- 把 A 客户的脸合成到 B 客户的身体上

**只做**：
- 抠图保留人像主体
- 替换背景（无人脸场景）
- 调色调（不改五官）

### 3.4 客户原片不入仓库

```
.gitignore 已规定：
mingshu-builder/clients/*/images/*.jpg
mingshu-builder/clients/*/images/*.jpeg
mingshu-builder/clients/*/images/*.png
```

⚠️ 仅 demo 客户（如 W 女士）的 images/ 可以为空（CSS fallback）。
