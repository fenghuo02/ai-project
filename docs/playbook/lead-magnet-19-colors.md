## ⚠️ 平台合规底线（先读这一段）

PDF 是公众号 / 小红书引流领取的成品物料，**违禁词检查比正文更严**——

- ❌ 算命 / 看相 / 八字 / 用神 / 调候 / 命格 / 格局 / 奇门 / 八门 / 死门 / 惊门

**正文（参考文案）**：

> ## 序 · 我们做的不是算命
>
> 这份 38 页 PDF 是渝山东方美学的 19 色本命色卡。
> 我们用东方传统的色彩理论体系——朱砂、石青、石绿、雌黄等矿物颜料的色相，配合季节、节气、生月——
> 推一个人天生最舒服的色彩调性。
>
> **不是算命**——我们不预测你的未来。
> **不是看相**——我们不评价你的过往。
> **是色彩**——我们告诉你：哪些颜色你穿了会发光，哪些颜色你身边布置会松下来。
>
> 翻这本 PDF 你会看到 19 个矿物色，每个色都有自己的应景节气、自己的适合调性。
> 读完，挑一个你最有感觉的颜色试穿一周，看看你和这个色之间发生什么。
>
> —— 渝山主理人 · 2026 春

每份 PDF 都内嵌一个独立编号，规则：

  - 例：`No.7B3F5C-20260301`

实现思路（**不在本 PR 范围**，属于 M2 工程化任务）：

```
1. 渲染基础 PDF（纯 38 页 + 占位水印 [USER_CODE]）
2. 公众号后台监听关键词「本命色」事件
   → 取 user OpenID 后 6 位 + 当前日期 → 拼接 user_code
   → 用 PyPDF2 把 user_code 渲染到 3 个固定位置 → 输出 individual.pdf
   → 上传到腾讯文档 / 七牛云 → 拿到外链
   → 通过公众号客服消息 API 推给 user
3. 用户拿到的 PDF 有自己的编号，转出去就是把自己实名了
```

- **首选**：腾讯文档（可设置「指定人可访问」+ 水印 + 可撤销）

## 八、与色谱母本的对接（开发者向）

PDF 内容不应**手抄** colors.yaml，应**程序生成**：

```
swatch-builder/palette/colors.yaml （母本）
        ↓
        ↓ 用 build.py 的渲染逻辑（已存在）
        ↓
        ↓ + 新写一个 lead-magnet 模板（templates/lead-magnet.html.j2）
        ↓
        ↓ → 渲染成 38 页 HTML
        ↓
        ↓ → weasyprint 转 PDF
        ↓
docs/playbook/output/lead-magnet-19-colors.pdf
```

> 实现路径建议（**不在本 PR 范围**）：
> 1. 在 `swatch-builder/templates/` 加 `lead-magnet.html.j2`
> 2. 在 `swatch-builder/build.py` 加一个 mode：`python3 build.py --lead-magnet`
> 3. 渲染时不读 client data.yaml，只读 colors.yaml + 这份大纲里的额外内容（P.03 序 / P.33 主理人手记 / P.34 案例 / P.35 SKU 表）
> 4. 输出到 `docs/playbook/output/`（gitignored）
>
> 这样**HEX 一致性自动保证**——和 mingshu-builder / swatch-builder 共用一套 palette 校验。
>
> 这个工程改造适合 PR E（M2 阶段），不在 PR D 范围内。
